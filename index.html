Phase 1: Define & Discover

Understand the context, define the problem, know the users, and set clear goals.

Brief Analysis: Break down the project request. Identify core objectives, constraints (time, budget, tech), target audience, key deliverables. Ask "Why?" repeatedly to uncover underlying needs. What assumptions are being made?

Stakeholder Interviews: Identify key stakeholders (product managers, engineers, marketing, sales, executives). Prepare questions focusing on business goals, success metrics, perceived user needs, technical feasibility, market context, and potential risks. Synthesize findings.

Based on initial understanding, articulate the core problem(s) this project aims to solve *from the user's perspective*. Use the format: "[Specific User Type] needs a way to [Perform Action/Achieve Goal] because [Insight/Pain Point/Barrier]". Ensure it's concise, actionable, and research-informed (even if initial research).

SMART Goals: Define Specific, Measurable, Achievable, Relevant, Time-bound goals for the project. What constitutes success?

HEART Framework (Optional): Consider measuring Happiness, Engagement, Adoption, Retention, Task Success. Define specific metrics for each relevant category.

Examples: Increase sign-up conversion by 15% in Q3. Reduce average task completion time for X by 20%. Achieve a System Usability Scale (SUS) score of 80+. Decrease support tickets related to Y by 25%.

Direct Competitors: Products offering very similar solutions to the same audience. Analyze their features, UX/UI, pricing, strengths, weaknesses, user reviews.

Indirect/Comparative: Products solving similar *problems* but perhaps differently or for a different audience. Look for inspiration in interaction patterns, onboarding, specific features.

Output: SWOT analysis (Strengths, Weaknesses, Opportunities, Threats), feature comparison matrix, UI/UX pattern observations, potential differentiators.

Objectives: What specific questions do you need to answer about user behaviors, needs, pain points, context? Link back to project goals and assumptions.

Participants: Define target user segments. Create screener questions to recruit appropriate participants (aim for 5-8 for qualitative interviews).

Methodology: Choose methods based on objectives (e.g., User Interviews, Contextual Inquiry, Surveys, Diary Studies, Analytics Review, Support Ticket Analysis).

Logistics: Timeline, budget (incentives?), tools, discussion guide/survey instrument.

Execution: Conduct the planned research activities ethically and effectively. Record sessions (with permission), take detailed notes.

Synthesis: Transcribe/organize raw data. Use methods like Affinity Diagramming (grouping notes by theme), identifying patterns, creating 'How Might We' questions based on insights. Summarize key findings, pain points, needs, motivations, and supporting quotes.

Create 1-3 fictional yet realistic representations of your primary user segments, *based entirely on research synthesis*. Include: Name, Photo/Archetype, Demographics (relevant ones), Goals & Needs (related to the product), Frustrations/Pain Points, Behaviors & Habits, Tech Savviness, Motivations, Bio/Scenario. Keep them focused and actionable.

Journey Map (As-Is): Visualize the persona's current step-by-step experience trying to achieve their goal without your solution. Map Phases > Actions > Thoughts > Feelings > Pain Points > Opportunities.

Empathy Map: Focus on a specific persona in a particular situation. Map what they Say, Think, Do, and Feel. Helps build deeper understanding and empathy.

If redesigning an existing product or analyzing competitors closely, evaluate against established usability heuristics (e.g., Nielsen's 10, Shneiderman's 8 Golden Rules). Document violations, provide examples, and rate severity (e.g., cosmetic, minor, major, catastrophic). Helps identify quick wins and areas needing significant change.

Phase 2: Ideate & Structure

Brainstorming solutions, prioritizing features, and defining the information flow and architecture.

Turn research insights and opportunities into actionable "How Might We..." (HMW) questions. Use these HMWs as prompts for brainstorming sessions (e.g., Crazy 8s, Round Robin, Worst Possible Idea). Generate a large quantity of diverse ideas without initial judgment. Capture everything visually (sketches, notes).

Group brainstormed ideas into logical features or epics. Evaluate and prioritize these features using established frameworks:

  • MoSCoW: Must Have, Should Have, Could Have, Won't Have (for this release).
  • Value vs. Effort Matrix: Plot features based on user value and implementation effort (focus on High Value/Low Effort first).
  • Kano Model: Categorize features as Basic, Performance, or Delighter.

Clearly define the scope for the Minimum Viable Product (MVP) â€“ the core set of features solving the main user problem.

For key features (especially MVP), break down the user's goal into specific, sequential tasks (Task Analysis). Create User Flow diagrams (flowcharts) illustrating the pathways users will take through the interface to complete these tasks. Show screens, decisions, actions, entry/exit points. Consider primary flows ("happy path") and potential alternative/error flows.

Create visual narratives (like comic strips) showing a persona using the product in a specific context to achieve their goal. Include panels showing the situation, user actions, interface interactions, and user's emotional response. Helps communicate the intended experience and context of use.

Define how information and features are organized, structured, and labeled to support usability and findability. Methods:

  • Content Inventory/Audit: List existing content (if redesigning).
  • Card Sorting (Open/Closed): Ask users to group content/features to understand their mental models.
  • Sitemap: Create a hierarchical diagram showing the structure of pages/screens and their relationships.
  • Navigation Design: Plan primary, secondary, and utility navigation systems.

Phase 3: Design & Prototype

Translating architecture and flows into tangible interfaces and interactive experiences.

Rapidly visualize different layout concepts for key screens identified in user flows and IA. Use paper sketches, whiteboards, or simple digital tools (e.g., Balsamiq, Excalidraw). Focus on basic structure, content placement, key UI elements, and flow. Don't worry about aesthetics. Iterate quickly based on peer feedback or self-critique.

Translate selected lo-fi concepts into more detailed digital wireframes using tools like Figma, Sketch, or XD. Use grayscale and placeholder content (or realistic if available). Focus on accurate layout, spacing, content hierarchy, UI element types (buttons, forms), basic information architecture, and interactions. Annotate where necessary to explain functionality.

Connect the mid-fidelity wireframe screens to create a clickable prototype simulating key user flows. Conduct usability testing with 3-5 target users. Focus on task completion, navigation effectiveness, clarity of information architecture, and identifying major usability issues *before* investing in visual design. Iterate on wireframes based on findings.

Define the visual identity and aesthetic direction. Gather inspiration (screenshots, photos, existing UIs) aligned with brand attributes and target audience preferences. Create mood boards to communicate look and feel. Explore different style tiles (combinations of color, typography, imagery, icons) for key UI elements.

Formalize the visual and interactive elements into a reusable system. Define: Color palettes (primary, secondary, semantic, neutrals), typography scale (headings, body, captions), iconography library, grid layout, spacing system (e.g., 8pt grid), reusable components (buttons, inputs, cards, modals) with variants and states (hover, focus, disabled). Prioritize consistency and accessibility (WCAG AA minimum).

Apply the established style guide and components to the iterated wireframes. Create pixel-perfect mockups for all necessary screens, including different states (empty, loading, error, success) and responsive variations. Focus on visual hierarchy, clarity, aesthetic appeal, and adherence to design principles. Use realistic content where possible.

Define how users interact with UI elements. Specify transitions between screens, micro-interactions (e.g., button feedback, loading spinners), animations, and gestures (for mobile). Build a high-fidelity prototype (using Figma, Protopie, etc.) that accurately simulates these interactions for testing and presentation.

Systematically review high-fidelity designs against Web Content Accessibility Guidelines (WCAG) AA standards. Check for sufficient color contrast, readable font sizes, adequate touch target sizes, clear focus indicators, logical content order (for screen readers), and appropriate labeling for interactive elements. Use automated tools and manual checks.

Phase 4: Test & Iterate

Validating the high-fidelity design with real users and refining based on empirical data.

Plan the main usability test for the high-fidelity prototype. Define: Clear objectives (e.g., Can users complete core tasks efficiently? Are interactions intuitive?), participant criteria (recruit ~5-8 users per segment), realistic task scenarios, key metrics (Task Success Rate, Time on Task, Error Rate, SEQ/SUS scores), testing environment (moderated/unmoderated, remote/in-person), and a detailed facilitator script.

Execute the planned usability tests. Ensure consistent facilitation if moderated. Encourage users to think aloud. Observe behavior closely and record quantitative metrics (success, time, errors) and qualitative feedback (comments, confusion points, body language). Collect satisfaction scores (SEQ, SUS).

Systematically analyze both quantitative (metrics) and qualitative (observations, quotes) data from testing. Use frameworks like Rainbow Spreadsheet or thematic analysis to identify recurring usability issues, friction points, and areas of success. Prioritize issues based on severity (impact on task completion, user frustration) and frequency. Use a severity scale (e.g., 0-4).

Implement design changes to address the prioritized usability issues in the high-fidelity designs and prototype. Document these iterations clearly (e.g., using "Before/After" comparisons, explaining the rationale based on test findings). If major changes were made, consider conducting quick validation testing on those specific areas with a few users to ensure the fix is effective.

If an A/B test was planned, implement the variations and launch the test using appropriate tools (e.g., Google Optimize, Optimizely). Monitor results until statistical significance is reached for the primary metric. Analyze the data to determine the winning variation and document the outcome and learnings.

Phase 5: Finalize & Present

Preparing final deliverables, documenting for handoff, and showcasing the project impact.

Perform meticulous final checks on all iterated UI screens across different breakpoints/states. Ensure pixel-perfect alignment, consistent application of the design system, final accessibility compliance, and overall visual refinement. Prepare and export all necessary graphical assets (icons, illustrations, images) in the required formats and resolutions for development.

Create comprehensive documentation for the development team to ensure accurate implementation. This typically includes: Final interactive prototype, link to the design system/style guide, detailed specs for components (measurements, typography, colors, states), interaction guidelines (animations, transitions), annotated mockups for complex screens, and user flow diagrams. Use tools like Zeplin, Figma Dev Mode, or create a dedicated spec document.

Craft a compelling story showcasing your design process, decisions, and the project's impact. Structure: 1. Overview (Hook, project summary, your role). 2. Problem (User needs, business context). 3. Goals (What success looked like). 4. Process & Approach (Outline steps taken). 5. Research (Key insights, personas). 6. Ideation & Design (Flows, wires, visual design evolution). 7. Testing & Iteration (Show findings and *how design changed* - crucial!). 8. Final Solution (Showcase polished UI/prototype). 9. Results & Impact (Connect back to goals, metrics). 10. Reflection (Learnings, challenges). Use high-quality visuals!

For real-world projects: After launch, collaborate with product/dev teams to monitor key metrics defined earlier (analytics, feedback channels, support tickets). Analyze performance against goals. Identify areas for future improvement or iteration based on real-world usage data. Plan next steps for the product roadmap.
